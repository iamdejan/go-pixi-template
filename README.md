# Go Pixi Template

This is a template repository for Go-based projects, but using Pixi as a version manager for Go.

## Prerequisites

Before running this program, ensure you have the following installed:

1. **Pixi** - A package manager for Rust and other languages. Install it by following the official guide at [https://pixi.prefix.dev/latest/](https://pixi.prefix.dev/latest/).

2. **Download Dependencies** - Once Pixi is installed, run the following command to download all project dependencies:
   ```bash
   pixi install
   ```

3. **Setup Pre-commit** - Install the pre-commit hooks to automatically run linting and build checks before push:
   ```bash
   pixi run pre-commit install --hook-type pre-push
   ```

   > **Note:** Re-run this command whenever `.pre-commit-config.yaml` is changed to update the hooks.

4. **(Optional) Install godoc-mcp** - Install the godoc-mcp server for Go documentation access:
   ```bash
   go install github.com/mrjoshuak/godoc-mcp@latest
   ```

   > **Note:** To install the MCP server, you need Go language installed in your computer, not just in Pixi virtual environment.

   > **Note:** This MCP server is pre-configured in `.kilocode/mcp.json` and will be automatically available to Kilo Code once installed.

## Run the Program

Use the following command to build and run the project:
   ```bash
   pixi run start
   ```

### Available Commands

| Command | Description |
|---------|-------------|
| `pixi run build` | Build the Go project |
| `pixi run start` | Build and run the program |
| `pixi run test` | Build and run all unit tests |
| `pixi run fmt` | Format the code using formatter in golangci-lint |
| `pixi run lint` | Run lint check (includes fmt check) |
| `pixi run lint-fix` | Auto-fix linting issues |

## Project Structure

```
Root/
├── pixi.toml               # Pixi workspace configuration
├── pixi.lock               # Pixi lockfile
├── go.mod                  # Go module manifest
├── go.sum                  # Go dependency checksums
├── LICENSE.txt             # Project license
├── README.md               # Project documentation
├── AGENTS.md               # Agent rules and guidelines
├── .gitignore              # Git ignore rules
├── .gitattributes          # Git attributes
├── .editorconfig           # Editor configuration
├── .golangci.yaml          # Go linter configuration
├── .pre-commit-config.yaml # Pre-commit hook configuration
├── .github/                # GitHub workflows
│   └── workflows/
│       ├── pr-pipeline.yaml
│       └── trunk-pipeline.yaml
├── .kilocode/              # Kilo Code assistant configuration
│   └── mcp.json            # MCP server configuration
├── main.go                 # Main entry point
└── message/                # Message package
    ├── hello.go            # Hello function implementation
    └── hello_test.go       # Unit tests for hello function
```

This section explains the purpose of each file in the repository:

### Root Directory Files

| File | Description |
|------|-------------|
| [`pixi.toml`](pixi.toml) | Pixi workspace configuration file that defines the project dependencies, tasks, and build settings. |
| [`pixi.lock`](pixi.lock) | Lockfile generated by Pixi to ensure reproducible builds across different environments. |
| [`go.mod`](go.mod) | Go module manifest that defines the module name, Go version, and external dependencies. |
| [`go.sum`](go.sum) | Checksums file that verifies the integrity of downloaded Go dependencies. |
| [`LICENSE.txt`](LICENSE.txt) | Project license file specifying the terms under which the code can be used. |
| [`AGENTS.md`](AGENTS.md) | Agent rules and guidelines for code generation, validation, and debugging. |
| [`README.md`](README.md) | Project documentation containing setup instructions, available commands, and project structure. |
| [`.gitignore`](.gitignore) | Git ignore rules specifying which files and directories should be excluded from version control. |
| [`.gitattributes`](.gitattributes) | Git attributes configuration for handling file types and line endings. |
| [`.editorconfig`](.editorconfig) | Editor configuration that ensures consistent coding styles across different editors and IDEs. |
| [`.golangci.yaml`](.golangci.yaml) | Configuration file for golangci-lint, which performs static analysis on Go code. |
| [`.pre-commit-config.yaml`](.pre-commit-config.yaml) | Configuration for pre-commit hooks that run checks before commits. |

### Source Files

| File | Description |
|------|-------------|
| [`main.go`](main.go) | Main entry point of the application that initializes and runs the program. |
| [`message/hello.go`](message/hello.go) | Package containing hello function that returns a greeting message. |
| [`message/hello_test.go`](message/hello_test.go) | Unit tests for the hello function in the message package. |

### Configuration Directories

| Directory | Description |
|-----------|-------------|
| [`.github/workflows/`](.github/workflows) | GitHub Actions workflow files for CI/CD pipelines. Contains `pr-pipeline.yaml` for pull request checks and `trunk-pipeline.yaml` for trunk-based development. |
| [`.kilocode/`](.kilocode) | Kilo Code assistant configuration. Contains `mcp.json` for MCP server setup. |
